# AdBeam VK Case Helper

Внутренний VK-бот для быстрого поиска кейсов AdBeam. MVP сделан простым: бот читает Excel, сохраняет кейсы в SQLite, ищет по тексту и фильтрам, отвечает карточками кейсов и работает только для разрешённых VK ID.

## Что умеет бот

- показывает главное меню кнопками VK;
- ищет кейсы по свободному запросу;
- фильтрует по нише, инструменту и цели;
- импортирует Excel-файл `data/cases.xlsx` в SQLite;
- ограничивает доступ через `ALLOWED_VK_IDS`;
- даёт статистику и обновление базы только админам;
- запускается локально или в Docker.

## Как устроен MVP

```text
app/
  main.py              # запуск бота через VK Long Poll
  config.py            # настройки из .env
  bot/                 # кнопки, handlers, доступы, форматирование
  cases/               # импорт Excel, поиск, модель кейса, правила продаж
  db/                  # SQLite-подключение и schema.sql
  utils/               # текстовая нормализация и логирование
tests/                 # минимальные sanity-тесты
data/                  # сюда кладём cases.xlsx, тут появится cases.db
```

## Создание VK-сообщества

1. Открой VK и создай сообщество: `Сообщества` → `Создать сообщество`.
2. Тип можно выбрать любой удобный, например `Бизнес`.
3. После создания открой `Управление` сообщества.
4. Перейди в `Сообщения` и включи сообщения сообщества.
5. Включи кнопку сообщений, чтобы сообществу можно было написать.

## Токен и Long Poll

1. В управлении сообществом открой `Работа с API`.
2. Создай ключ доступа.
3. Для MVP нужны права на сообщения сообщества.
4. Скопируй токен и сохрани его только в `.env`.
5. Открой `Long Poll API`.
6. Включи Long Poll.
7. Выбери актуальную версию API.
8. В событиях включи `Входящие сообщения` / `message_new`.

Для MVP используется Long Poll, поэтому внешний сервер и Callback API не нужны.

## Как узнать свой VK ID

Самый простой путь:

1. Открой свою страницу VK в браузере.
2. Если адрес вида `vk.com/id123456789`, число после `id` и есть VK ID.
3. Если адрес красивый, можно открыть любое сообщение/комментарий или использовать сервисы определения VK ID.

Твой ID нужно добавить в `ADMIN_VK_IDS` и `ALLOWED_VK_IDS`.

## Настройка `.env`

Скопируй пример:

```bash
copy .env.example .env
```

Заполни `.env`:

```env
VK_GROUP_TOKEN=токен_сообщества
ADMIN_VK_IDS=123456789
ALLOWED_VK_IDS=123456789,987654321
CASES_XLSX_PATH=data/cases.xlsx
DB_PATH=data/cases.db
LOG_LEVEL=INFO
```

`.env` уже добавлен в `.gitignore`, токен не должен попадать в Git.

## Excel с кейсами

Положи файл с кейсами сюда:

```text
data/cases.xlsx
```

Поддерживаются листы вроде:

- `Advantage ENG`
- `Adbeam RU`
- `Презентация RU`
- `Upwork`

Ожидаемые колонки могут немного отличаться по регистру:

- `Проект`
- `Название кейса`
- `Ссылка на кейс`
- `Ниша`
- `Ниша доп`
- `Цель`
- `Инструмент`
- `ГЕО` или `Гео`
- `Аудитория`
- `Где лежат`

Если база пустая и файл `data/cases.xlsx` есть, бот импортирует его при первом запуске.

## Локальный запуск без Docker

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m app.main
```

Отдельный импорт Excel:

```bash
python -m app.cases.importer
```

## Запуск через Docker

Сначала создай `.env` и положи Excel в `data/cases.xlsx`.

```bash
docker compose up --build -d
```

Посмотреть логи:

```bash
docker compose logs -f bot
```

Остановить:

```bash
docker compose down
```

На сервере обычно достаточно скопировать проект, создать `.env`, положить Excel в `data/cases.xlsx` и запустить `docker compose up --build -d`.

## Как проверить бота

1. Запусти бота.
2. Напиши сообщение сообществу VK.
3. Отправь `/start`.
4. Проверь, что появилось меню с кнопками.
5. Нажми `🔎 Найти кейс`.
6. Введи запрос, например `парфюм директ`.
7. Проверь кнопки `🛠 Кейсы по инструменту` и `🎯 Кейсы по цели`.
8. Под админским VK ID проверь `📊 Статистика базы` и `🔄 Обновить базу`.

## Команды

- `/start` — главное меню;
- `/help` — помощь;
- `/case <запрос>` — поиск кейсов;
- `/filters` — подсказка по фильтрам;
- `/stats` — статистика, только admin;
- `/reload_cases` — обновить базу, только admin.

## Как обновить Excel

1. Замени файл `data/cases.xlsx`.
2. В VK нажми `🔄 Обновить базу` под админом.
3. Или выполни:

```bash
python -m app.cases.importer
```

В Docker:

```bash
docker compose exec bot python -m app.cases.importer
```

## Как добавить новый тип кнопок

1. Добавь текст кнопки в `app/bot/keyboards.py`.
2. Добавь состояние в `app/bot/states.py`, если бот должен ждать следующий ввод.
3. Обработай кнопку в `app/bot/handlers.py`.
4. Если это фильтр по новому полю, добавь поле в `FIELD_MAP` в `app/cases/search.py`.

## Тесты

```bash
pytest
```

Тесты проверяют импорт Excel, поиск и простые правила блока «Как использовать в продаже».

Excel читается через `openpyxl`, поэтому на сервере не нужны тяжёлые зависимости вроде `pandas` и `numpy`.

## Частые ошибки

`VK_GROUP_TOKEN не задан`

Проверь, что создан `.env` и в нём есть токен.

`Это внутренний бот AdBeam. Доступ ограничен.`

Твой VK ID не добавлен в `ALLOWED_VK_IDS`.

Бот запустился, но не отвечает

Проверь, что в VK включены сообщения сообщества, Long Poll API и событие `message_new`.

Поиск ничего не находит

Проверь, что `data/cases.xlsx` лежит на месте и импорт прошёл без ошибок. Под админом нажми `📊 Статистика базы`.

Docker не видит Excel

Проверь, что файл лежит в локальной папке `data/`. Она монтируется в контейнер как `/app/data`.
